Softeneers Framework — Documentation

Start here. Suggested reading order:

  1. ARCHITECTURE.md — what the framework is, the monorepo

layout, the package-vs-template distinction, conventions.

  1. ROADMAP.md — the sprints, the MVP, and current status

(the source of truth for "what's done").

  1. CLI-SPEC.md — the create-softeneers-app contract:

prompts, flags, and the copy/transform algorithm.

  1. PACKAGES.md — the contract for each @softeneers/* package.
  2. DECISIONS.md — authoritative decisions / tiebreakers.

When two docs seem to conflict, this file wins.

  1. PUBLISHING.md — how the packages get to npm.

Related, at the repo root: ../README.md (overview), ../CONTRIBUTING.md (setup + conventions), ../TODO.md (original intent), ../PROJECT-BENCHMARK.md (latest maturity benchmark), ../CLAUDE.md (agent entry contract). Each doc has a committed, human-readable .html companion beside it (e.g. ARCHITECTURE.html next to ARCHITECTURE.md), with a browsable index.html at the repo root. These are generated from the Markdown by apps/docs (npm run build) — never hand-edit them. The deployed, public marketing+docs site is live at <https://softeneers-landing.vercel.app> (source in apps/landing).

Conventions

  • .md files are the AI-optimized source of truth; their .html companions are

the generated human view (see DECISIONS.md D-07).

  • Every relative link in these docs must resolve (checked in CI via the docs

link test).

  • Status claims live in ROADMAP.md; other docs link to it rather than

duplicating status.
